# Gates

Every major phase in Track 02 starts with a gate.

Each gate must freeze:

- canon review notes
- scope and acceptance criteria
- contracts and types
- art manifest
- validation plan
- open TBDs elevated or closed

No phase starts implementation without one approved gate document.

## Supersession Rule

Approved gates preserve the decision context from the moment they were opened.

When a later gate changes product posture, the later gate wins for future work. In particular, F10 and F11 supersede earlier frontend/PvP framing: Campaign is primary, Extras are secondary, and Arena PvP / Private Duel is hidden from normal public navigation until a future gate explicitly promotes, prototypes, or removes it.
