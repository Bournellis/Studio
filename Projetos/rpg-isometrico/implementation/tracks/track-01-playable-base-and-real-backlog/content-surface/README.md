# Content Surface

This directory is the active Godot-first home for content-package decisions inside Track 01.

Use it to separate:

- the exact content currently authored in `definitions/`
- the rules for recording future Godot content packages
- the currently adopted implementation-local candidate lane for active expansion work

Do not use this directory to redefine shared canon.

Shared canon still owns:

- product identity
- race territories
- the loadout contract
- release-horizon direction

This directory owns:

- the exact implemented Godot content baseline
- Godot-local naming for package candidates
- the rules for deciding what stays implementation-local versus what is promoted to canon later
