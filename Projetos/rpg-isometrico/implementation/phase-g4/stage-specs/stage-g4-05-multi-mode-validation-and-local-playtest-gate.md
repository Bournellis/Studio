# Stage G4-05 - Multi-Mode Validation And Local Playtest Gate

## Goal

Close Phase G4 by proving that Godot now supports a believable local multi-mode base instead of only a single accepted combat slice.

## Required Outcome

- headless validation and GUT cover Arena, Survival, and Boss baseline flows
- manual smoke guidance exists for the multi-mode local runtime
- frontend routing, mode entry, in-match flow, results, and return are smokeable without guesswork
- the Godot project can be judged as a believable local multi-mode base
- the next checkpoint can choose between content reformulation, Steam prep, or campaign prep without reopening mode-foundation questions

## Scope

- validation expansion
- smoke checklist expansion
- local multi-mode regression pass
- checkpoint handoff notes

## Non-Goals

- new runtime features beyond what the prior stages require
- release-build preparation
- public playtest operations
- Steam integration
