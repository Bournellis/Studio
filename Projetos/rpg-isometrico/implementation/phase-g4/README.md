# Historical Validation Phase G4 - Shared Solo Base Expansion

Phase G4 is preserved as the fourth closed phase in the Godot validation cycle.

## Goal

Expand the proven `frontend -> arena -> result/return` slice into a shared solo runtime base that can carry:

- Arena
- Survival
- Boss Mode

## Focus

- shared mode foundation and frontend routing
- Survival playable baseline
- Boss playable baseline
- shared combat-shell and results parity across solo modes
- multi-mode validation and local playtest readiness

## Non-Goals

- Steam or online runtime
- co-op production support
- mobile controls
- campaign slice
- broad official content reformulation
- new race breadth

## Operational Files

- `../execution-log.md`
- `current-status.md`
- `implementation-map.md`
- `progress-log.md`
- `stage-specs/`

## Exit State

Phase G4 should leave Godot with a credible multi-mode solo runtime base that supports deliberate planning for later content reformulation, Steam preparation, or campaign work.

Current handoff: `checkpoint-g4-local-multi-mode-base-acceptance.md` is now the acceptance gate for that decision.
