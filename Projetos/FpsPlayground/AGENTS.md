# AGENTS.md

This file governs agent behavior for `Projetos/FpsPlayground`.

## Project Role

`FpsPlayground` is an official implementable Godot project for PC Windows editor-first first-person gameplay experiments.

It was split from the former `Projetos/FpsShooter` project on 2026-06-10. The football/TPS work now lives in `Projetos/JogoDaCopa`.

Current identity:

- first-person playground tech probe;
- local `Arena Shooter` 1x1 mode against a bot;
- `Duel Pit V2` arena baseline with rifle, Plasma Bolt, pickups, jump pads, bot pressure and knockback;
- PC Windows editor-first only;
- no football/TPS minigame scope.

## Godot Rule

- Engine: Godot `4.6.2-stable`
- Language: GDScript only
- Tests: GUT `9.6.0`
- Current platform: PC Windows editor-first only
- Scene rule: scenes are generated by `tools/bootstrap_scene_generator.gd`; do not hand-edit generated `.tscn` files as raw text.
- Prefer runtime-generated primitive meshes and collision for prototype work.

## Architecture Rule

- `autoloads/`: input/bootstrap only.
- `gameplay/combat/`: health, damage, knockback and shared combat body behavior.
- `gameplay/player/`: FPS movement, camera, aiming, rifle shooting and alt-fire requests.
- `gameplay/bot/`: arena bot movement and shooting behavior.
- `gameplay/arena/`: arena rule helpers.
- `modes/menu/`: FpsPlayground mode entry.
- `modes/arena/`: arena assembly, round state and hit/projectile resolution.
- `modes/shared/`: shared runtime primitive creation.
- `presentation/hud/`: arena HUD.
- `presentation/feedback/`: transient primitive/audio feedback.
- `tools/`: scene generation and validation.
- `tests/`: GUT coverage for input, scene assembly, arena combat and helper contracts.
- `docs/`: local product/technical references.
- `implementation/`: status and track records.
